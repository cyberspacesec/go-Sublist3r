go-Sublist3r 文档

欢迎使用go-Sublist3r文档。本文档将帮助您了解如何安装和使用go-Sublist3r进行子域名枚举。

go-Sublist3r是一个用Go语言实现的Sublist3r子域名枚举工具包装器。它提供了一种使用Docker运行Sublist3r的便捷方法，无需在系统上直接安装Python依赖项。

提示: 如果您对API集成感兴趣，请查看API文档。

安装

前提条件

  • Go 1.16或更高版本
  • Docker (可选，如果不可用会使用模拟模式)

从源代码构建

git clone https://github.com/cyberspacesec/go-Sublist3r.git
cd go-Sublist3r
make build

全局安装

要将go-Sublist3r安装到系统中，使其可以在任何目录中使用，请运行： 

make install

这将把二进制文件复制到/usr/local/bin/目录中。

使用方法

基本用法

最基本的用法是指定一个要扫描的域名： 

go-sublist3r scan -d example.com

这将为example.com域名枚举子域名并在控制台上显示结果。

保存结果到文件

使用-o参数将结果保存到文件： 

go-sublist3r scan -d example.com -o results.txt

使用暴力破解模块

添加-b标志启用暴力破解模块： 

go-sublist3r scan -d example.com -b

指定扫描端口

使用-p参数可以对发现的子域名扫描指定的TCP端口： 

go-sublist3r scan -d example.com -p 80,443

Docker镜像管理

go-Sublist3r会自动处理Docker镜像，但您也可以使用以下命令手动管理镜像：

从Docker Hub拉取镜像

go-sublist3r pull-docker-image

构建本地Docker镜像

go-sublist3r build-docker-image

提示: 默认情况下，程序会优先尝试从Docker Hub拉取镜像，仅当拉取失败时才会尝试构建本地镜像。

高级选项

完整的命令选项

  • -d, --domain: 要枚举子域名的域名（必需）
  • -b, --bruteforce: 启用subbrute暴力破解模块
  • -p, --ports: 扫描找到的子域名上的指定tcp端口
  • -v, --verbose: 启用详细模式并实时显示结果
  • -t, --threads: 用于subbrute暴力破解的线程数（默认：30）
  • -e, --engines: 指定以逗号分隔的搜索引擎列表
  • -o, --output: 将结果保存到文本文件
  • -n, --no-color: 不带颜色输出

指定搜索引擎

使用-e参数可以指定要使用的搜索引擎： 

go-sublist3r scan -d example.com -e "baidu,yahoo,google"

调整线程数

使用-t参数可以调整暴力破解模块使用的线程数： 

go-sublist3r scan -d example.com -b -t 50

API服务器

go-Sublist3r提供了一个RESTful API，允许通过HTTP请求进行子域名枚举。

启动API服务器

go-sublist3r api --port 8080 --workers 5 --capacity 100

API服务器参数

  • --port: API服务器监听的端口号 (默认: 8080)
  • --workers: 处理扫描请求的工作线程数 (默认: 5)
  • --capacity: 最大并发扫描请求数 (默认: 100)
  • --api-key: API密钥，用于请求认证（留空表示不启用认证）

使用API密钥保护API

要启用API密钥认证，可以在启动API服务器时指定密钥： 

go-sublist3r api --port 8080 --api-key "your-secret-key"

安全提示: 在生产环境中使用API时，强烈建议启用API密钥认证以保护API不被未授权访问。
